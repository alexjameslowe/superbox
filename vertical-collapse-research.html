Vertical collapse
We have need for a "collapse" mode for vertical and horizontal containers, and by that we mean a container where it collapses to a frugal width. sb-v and sb-h containers usually take up 100% width.

But there are cases where we just want a containers to only size horizontally based on their contents. Like for instance if there's some little info box that's supposed to float over the screen, like a drop-down menu or something like that. The whole point of this framework is that as much as possible, we get away from worrying about the sizes of containers, and we care more about how they take up space and how they stack.

So here, we have a vertical container that only takes up width frugally. This will necessarily break the concept of greedy and frugal, which is a side-effect of this technique. We also get an extra degree of freedom, which is that we can designate one of the rows to be the width-setter. So below, the middle element sets the width of all the other elements, even though it's not actaully the longest/biggest content. This is nice because we can acheive a good layout that responds to a given content-area and doesn't require any repulsive widths or wrapper elements.

This relies on table display settings. It's a strange technique. All those strange table-displays come in handy for situations like this. This effect is not possible to do with flexbox. As a result of this reliance and table-display styles, we can use padding.

Weird things: We can use margin too, although there's a little bug with safari in the below example. Notice that we need extra right-padding to make it behave in that browser. And it the outer wrapper disobeys the margin-top style.
